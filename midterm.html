3. Midterm Activities

Stack vs. Heap Memory - Diagrammatic Representation

Through handwritten diagrams and tables, I detailed the conceptual mapping of Java variables, distinguishing between reference types and primitive types and visualizing where each is stored in memory (Stack for local variables, Heap for objects).

Learnings:

Understanding memory management (Stack/Heap) is critical for optimizing Java application performance and preventing memory leaks.

Memory Impact of Object Creation and Reassignment

Building on Part 1, I documented the sequence of a program where a new student object 's1' is created, a second object 's2' is created, and then the reference 's3' is reassigned to point to the same object as 's2'. A final table tracks the updated values in the Heap.

Learnings:

Learned how pointers (references) in Java work, specifically that a reassignment change (like s3.name = "Ana") will affect all references to that same object, demonstrating polymorphism and object reference management.

Activity: Secure ATM Terminal System

Decision Structures | Methods

This comprehensive Java application simulates an ATM terminal, handling secure login, balance checking, deposits, and multi-step withdrawals with account lockouts. It uses nested if-else structures to validate passwords and balance limits.

Learnings:

Implementing methods with returns to modularize code (checkBalance, deposit). Learns to manage state within an object to prevent illegal actions (withdrawing more than the limit or balance).

Activity: Tuition Payment System with Scholar Discounts

A system that applies different logic for scholar vs regular students, calculates new tuition, and processes payments until the balance is zero, or a set number of incorrect attempts are reached.

Learning: Using if-else chains to create complex logic trees based on multiple inputs and handling simple lockout conditions.

Activity: Simple Student Expense Tracker

Calculates student expense summaries from inputs, compares them against a set budget, and uses basic decision structure logic to provide an overall budget status (e.g., 'Within budget!').

Learning: Practical application of decision structures (`if`) to provide meaningful, state-based feedback to the user.

Through hands-on coding (file assignment.java), I explored the difference between conditional and bitwise operators, as well as the syntax of the ternary operator. The provided VS Code screenshot captures both the source code and the terminal output simultaneously, serving as rich evidence.
